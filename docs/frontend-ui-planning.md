# HiveLogix 前端界面规划

> 基于"无人机-卡车-仓库-充换电站"四元协同物流调度系统的前端 UI 设计规划
> 文档生成日期：2026-03-28

---

## 一、全局导航架构 (Navigation Tree)

```plaintext
HiveLogix_Frontend_App
├── 📊 调度性能大屏 (Dashboard)         // 全局KPI、调度性能统计与预测
├── 🗺️ 实时指挥中心 (Dispatch Center)   // 核心工作台：地图可视化、实时调度干预
├── 📦 订单与任务管理 (Order & Task)    // 订单梳理、任务下发、路径清单
├── 🛸 载具管理调度 (Fleet Management)  // 卡车状态、无人机序列监控
├── 🏗️ 基础设施配置 (Infrastructure)    // 仓库网点、充换电站网络设置
└── ⚙️ 仿真与动态场景 (Simulation Box)  // 动态订单生成规则、地图区域选取、算法约束调优
```

---

## 二、核心模块组件拆解 (Component Data Tree)

### 1. 🗺️ 实时指挥中心 (Dispatch Center)

这是系统的"大脑"，融合了地图可视化与实时决策面板。

```plaintext
Dispatch_Center_View
├── 🎛️ 左侧：实时状态面板 (Real-time Status Panel)
│   ├── 动态订单流入队列 (滚动列表, 高亮临近 t_deadline 的订单)
│   ├── 告警中心 (缺货预警 / E_arrive 低电量预警 / 超时预警)
│   └── 快速决策卡片 (当模式E被触发时，弹出算法推荐的中继路径供确认)
│
├── 🗺️ 中央：多维 GIS 地图交互区 (Map Visualization)
│   ├── 地图控制工具栏 (底图切换、图层显隐控制、时间加速/回放)
│   └── 动态图层组 (Layers)
│       ├── 静态设施层: 仓库图标 (无限库存)、充换电站图标 (带排队指示灯)
│       ├── 地面动态层: 卡车实时位置点、卡车历史与规划未来轨迹线 (贴合路网)
│       ├── 空中动态层: 无人机实时位置、无人机 3D 飞行轨迹线 (按模式A-E区分颜色)
│       ├── 任务热力层: 客户订单位置分布 (静态预订单 vs 动态散单)
│       └── 能量拓扑层: 选中无人机时，悬浮显示路径节点上的 E_arrive 和 E_depart
│
└── 📋 右侧：高优实体详情抽屉 (Entity Detail Drawer)
    ├── 卡车视图: 当前载货量 (进度条)、搭载无人机列表、下一汇合节点预测
    ├── 无人机视图: 电池余量 E_max 占比、载重状态、当前履约模式 (A/B/C/D/E)
    └── 电站视图: 当前服务架次、预计排队时间、换电/充电模式标识
```

### 2. ⚙️ 仿真与动态场景设置 (Simulation & Algorithm Box)

满足"地图选取"和"动态订单设置"的需求入口。

```plaintext
Simulation_Config_View
├── 🗺️ 地图选取与环境构建 (Map ROI Selector)
│   ├── 多边形框选工具 (在地图上划定 4000m x 4000m 仿真服务区)
│   ├── 禁飞区/路网障碍物绘制工具
│   └── 站点快速部署 (拖拽生成仓库 Depot 和电站 S_i)
│
├── 📦 动态订单生成器 (Dynamic Order Generator)
│   ├── 空间分布设置 (均匀分布 / 泊松分布 / 商业区热力聚集)
│   ├── 时间分布设置 (接单频率、高峰期涌入率)
│   └── 订单属性边界: 货物重量 q_i 范围、软时间窗截止 t_deadline 宽容度
│
└── 🧠 调度求解器参数配置 (Solver Parameters)
    ├── 容量约束: 卡车容量 C_truck、无人机载重 C_drone、电量 E_max
    ├── 惩罚权重调整: 超时罚金系数 vs 耗电成本系数权重拉杆
    └── 履约模式开关: 可单独开启/关闭 模式C(仓直递)、模式D(空投补货)、模式E(多跳)
```

### 3. 📊 调度性能大屏 (Dashboard & Performance)

用于量化算法和调度策略的优劣。

```plaintext
Dashboard_View
├── 🏆 核心全局 KPI (Top Banner)
│   ├── 综合任务完成率 (%)
│   ├── 准时送达率 (On-time Ratio)
│   ├── 平均订单延迟 (Avg Delay, 单位: 分钟)
│   └── 总体能耗成本 (Total Energy / 续航消耗)
│
├── 📈 运筹模式构成图 (Fulfillment Mode Distribution)
│   └── 饼图/漏斗图: 模式A / B / C / D / E 的触发频次与占比
│
├── ⚡ 能量与中继效率监控 (Energy & Relay Analytics)
│   ├── 充换电站排队拥堵热力图 (识别网络瓶颈)
│   └── 平均换电滞留时间 (t_depart - t_arrive)
│
└── ⏱️ 时效与惩罚成本趋势 (Cost & Penalty Timeline)
    └── 双折线图: 随时间推移的"订单涌入量"与"累计超时惩罚成本"
```

### 4. 📦 订单与任务管理 (Order & Task Management)

将抽象的算法输出转化为可读的物流运单清单。

```plaintext
Task_Management_View
├── 🗂️ 订单全量表 (Master Order List)
│   ├── 属性列: 订单ID、类型(静态/动态)、客户坐标、重量 q_i
│   ├── 时间列: t_release、t_deadline、实际送达时间
│   └── 状态列: 待分配 / 正在规划 / 履约中 (附带履约模式标签) / 已送达 / 已超时
│
└── 🚀 派送任务拆解视图 (Task Route Breakdown)
    └── 树形表格 (Tree Table): 详细展示一架无人机/卡车的完整流转
        ├── [父节点] 任务ID - 归属对象 - 总预计耗时
        ├── [子节点 1] 起飞阶段: 关联节点 (卡车/仓库)、初始电量
        ├── [子节点 2] 投递阶段: 目标客户位置、卸载动作
        ├── [子节点 3] (若触发模式E) 中继阶段: 目标电站、E_arrive预估、等待+换电耗时
        └── [子节点 4] 汇合阶段: 追赶卡车/返仓、最终剩余电量安全校验
```

### 5. 🏗️ & 🛸 资源网格配置 (Infrastructure & Fleet)

静态与动态资源的数字化映射。

```plaintext
Resource_Management_Views
├── 🏗️ 站点网格化管理 (Infra Setup)
│   ├── 充换电站列表: 坐标、最大并行容量、补能模式 (充电 T_charge=α·ΔE 或 换电 τ_swap)
│   └── 仓库网点列表: 固定坐标、当前备用无人机数量
│
└── 🛸 载具资产中心 (Fleet Registry)
    ├── 卡车序列: 编号、当前分配路径策略、最大库容量
    └── 无人机序列: 编号、最大电池容量、当前健康度、绑定母港 (某个卡车或仓库)
```
